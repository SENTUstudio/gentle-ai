## When to Use

- When source data already exists in Glue Catalog (refined zone)
- When multiple source tables need to be joined
- When complex SQL CTEs are needed for transformation logic
- When full table reload is acceptable (vs incremental S3)
- During `sdd-design` phase for Glue Catalog-based ETL approach

## Critical Patterns

### Reading from Glue Catalog

Use `create_dynamic_frame.from_catalog()` for Glue-native reading:

```python
dynamic_frame = glueContext.create_dynamic_frame.from_catalog(
    database=db_name,
    table_name=table_name,
    transformation_ctx=f"lectura_{table_name}",
)
df = dynamic_frame.toDF()
df.createOrReplaceTempView(table_name)
```

### Multi-Table Reading Pattern

Read all source tables into temporary views:

```python
CATALOG_TABLES = [
    (DB_SOURCE, "orders"),
    (DB_SOURCE, "customers"),
    (DB_SOURCE, "products"),
]

for db_name, table_name in CATALOG_TABLES:
    dynamic_frame = glueContext.create_dynamic_frame.from_catalog(
        database=db_name,
        table_name=table_name,
        transformation_ctx=f"lectura_{table_name}",
    )
    df = dynamic_frame.toDF()
    df.createOrReplaceTempView(table_name)
```

### SQL CTEs with WITH...AS

Use readable multi-step CTEs for complex transformations:

```sql
WITH step_1_filtered AS (
    SELECT * FROM orders WHERE status = 'ACTIVE'
),
step_2_enriched AS (
    SELECT
        o.*,
        c.customer_name,
        c.region
    FROM step_1_filtered o
    JOIN customers c ON o.customer_id = c.customer_id
),
step_3_aggregated AS (
    SELECT
        region,
        COUNT(*) as order_count,
        SUM(amount) as total_amount
    FROM step_2_enriched
    GROUP BY region
)
SELECT * FROM step_3_aggregated
```

### Output Pattern: Coalesce(1) + Overwrite

For single-file output (Quicksight compatibility):

```python
s3_output_path = f"s3://{BUCKET}/<table_name>/"
df_final.coalesce(1).write.mode("overwrite").parquet(s3_output_path)
```

### Schema Extraction Phase
When extracting schema (Phase 1):
- **Comment out** write sections
- **Add** `df_final.printSchema()`
- Document column types for INFRA table definition

## Code Examples

### Glue Job: ETL from Glue Catalog

```python
# ============================================================================
# Glue Job: ETL_<table>_carga
# source: Glue Catalog
# author: generated by gentle-ai
# ============================================================================

import sys
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from awsglue.dynamicframe import DynamicFrame
from awsglue.job import Job
from awsglue.utils import getResolvedOptions

# Configuration
DB_SOURCE = "toyota_chile_refined"
DB_DEST = "toyota_chile_refined"
BUCKET = "toyota-chile-refined"

# Table configurations
CATALOG_TABLES = [
    (DB_SOURCE, "orders"),
    (DB_SOURCE, "customers"),
    (DB_SOURCE, "products"),
]

# ============================================================================
# INIT
# ============================================================================
args = getResolvedOptions(sys.argv, ["JOB_NAME"])
sc = SparkContext.getOrCreate()
glueContext = GlueContext(sc)
spark = glueContext.spark_session
job = Job(glueContext)
job.init(args["JOB_NAME"], args)

# ============================================================================
# READ FROM CATALOG
# ============================================================================
for db_name, table_name in CATALOG_TABLES:
    dynamic_frame = glueContext.create_dynamic_frame.from_catalog(
        database=db_name,
        table_name=table_name,
        transformation_ctx=f"lectura_{table_name}",
    )
    df = dynamic_frame.toDF()
    df.createOrReplaceTempView(table_name)
    print(f"Loaded {table_name}: {df.count()} rows")

# ============================================================================
# TRANSFORM (SQL with CTEs)
# ============================================================================
query = """
WITH filtered_orders AS (
    SELECT *
    FROM orders
    WHERE status = 'ACTIVE'
      AND order_date >= '2024-01-01'
),
enriched_with_customer AS (
    SELECT
        o.order_id,
        o.order_date,
        o.amount,
        c.customer_name,
        c.region,
        c.segment
    FROM filtered_orders o
    JOIN customers c ON o.customer_id = c.customer_id
),
enriched_with_product AS (
    SELECT
        e.*,
        p.product_category,
        p.product_name
    FROM enriched_with_customer e
    JOIN products p ON e.product_id = p.product_id
),
final_output AS (
    SELECT
        order_id,
        order_date,
        customer_name,
        region,
        segment,
        product_category,
        product_name,
        CAST(amount AS DECIMAL(18,2)) AS amount
    FROM enriched_with_product
)
SELECT * FROM final_output
"""

df_final = spark.sql(query)

# ============================================================================
# WRITE TO S3 — Commented for schema extraction phase
# ============================================================================
# s3_output_path = f"s3://{BUCKET}/<table_name>/"
# df_final.coalesce(1).write.mode("overwrite").parquet(s3_output_path)

# ============================================================================
# SCHEMA EXTRACTION (uncomment for Phase 1)
# ============================================================================
df_final.printSchema()
print("Schema extraction complete")

# ============================================================================
# COMMIT — Always call job.commit() before exiting
# ============================================================================
job.commit()
```

## Commands

### Run ETL Locally (Development)

```bash
export DB_SOURCE="toyota_chile_refined"
export DB_DEST="toyota_chile_refined"
export BUCKET="toyota-chile-refined"

python glue_etl.py
```

### Check Glue Catalog Tables

```bash
# List tables in database
aws glue get-tables --database-name toyota_chile_refined

# Get table schema
aws glue get-table --database-name toyota_chile_refined --table-name orders
```

### Validate SQL in Athena

```bash
# Preview transformation without writing
aws athena start-query-execution \
    --query-string "WITH filtered AS (SELECT * FROM orders LIMIT 1) SELECT * FROM filtered"
```
